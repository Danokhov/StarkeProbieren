# 📊 Инструкция по системе прогресса

## 🎯 Общие принципы

### Структура приложения
Приложение состоит из **11 модулей** + 3 специальных раздела:
- **Модули 1-11**: Тематические уроки по сильным глаголам
- **БАЗА**: База всех сильных глаголов с ассоциациями
- **КВИЗ**: Тестирование знаний
- **ПОВТОР**: Интервальное повторение карточек

---

## 📈 Прогресс модулей

### Как рассчитывается прогресс модуля?

Каждый модуль состоит из **разделов (секций)**. Прогресс модуля = среднее арифметическое прогресса всех разделов.

**Формула:**
```
Прогресс модуля = (Сумма прогрессов всех разделов) / (Количество разделов) × 100%
```

### Типы разделов в модулях

#### 1. 📹 **Видео** (Ассоциации, Теория, История банды)
- **Условие завершения**: ⚠️ **Открытие раздела с видео**
- **Прогресс**: 0% или 100%
- **Как отслеживается**: 
  - Раздел автоматически отмечается как завершенный при **клике на карточку** раздела
  - Фактический просмотр видео до конца **НЕ отслеживается технически**
  - Это сделано для упрощения UX (пользователь сам решает, смотреть видео или нет)
- **Правило**: Достаточно один раз открыть раздел

#### 2. 🗂️ **Карточки** (обычные и в 3-х формах)
- **Условие завершения**: Просмотр всех карточек в сессии
- **Прогресс**: 0% или 100%
- **Правило**: Нужно дойти до конца колоды (ответы "Знаю"/"Не знаю" не влияют на прогресс модуля)

#### 3. 🎵 **Мантры** (Модули 1-2)
- **Условие завершения**: ✅ **Отметить все мантры как "Знаю"**
- **Прогресс**: 0% или 100%
- **Как работает**:
  - Мантры работают как **карточки** с аудио
  - Есть кнопка 🔊 для воспроизведения (не обязательно нажимать)
  - Карточка переворачивается по клику (DE ⟷ RU)
  - Внизу две кнопки: **"Трудно"** ❌ и **"Знаю"** ✅
  - Раздел завершается только когда **ВСЕ мантры отмечены как "Знаю"**
- **Правило**: Пройти все мантры и отметить их зелёной кнопкой "Знаю"

#### 4. 🎵 **Мантры** (Модули 3-11) - *Особая логика*
- **Разделение на 3 формы**:
  - **Präsens** (настоящее время)
  - **Präteritum** (прошедшее время)
  - **Partizip II** (причастие прошедшего времени)
- **Условие завершения**: ✅ **Отметить все мантры как "Знаю" во всех трёх формах**
- **Как работает**:
  - Переключатель времени сверху: **Präsens** / **Präteritum** / **Partizip 2**
  - Мантры работают как **карточки** с аудио
  - Кнопка 🔊 для воспроизведения (не обязательно)
  - Внизу две кнопки: **"Трудно"** ❌ и **"Знаю"** ✅
  - Нужно пройти **ВСЕ мантры** в **КАЖДОЙ** из трёх форм
- **Прогресс**: Каждая форма = отдельный подраздел
- **Правило**: Отметить все мантры как "Знаю" в Präsens, Präteritum и Partizip II

#### 5. ✏️ **Пробелы в мантрах** (Модули 1-2)
- **Условие завершения**: ✅ **Все упражнения выполнены ПРАВИЛЬНО (100%)**
- **Прогресс раздела**: (Количество правильных ответов) / (Всего упражнений) × 100%
- **Как работает**:
  - ✅ Правильный ответ → сохраняется как `правильно`
  - ❌ Неправильный ответ → сохраняется как `неправильно`
  - **Прогресс отображается постепенно**: 30%, 50%, 80% и т.д.
  - **НО раздел считается завершенным** только при 100% правильных!
  - Галочка ✅ появляется **только** когда все упражнения без ошибок
- **Правило**: 
  - Детальный прогресс **сохраняется** для каждого упражнения
  - Раздел **не завершается** (0% для модуля), пока не достигнут 100%
  - Можно вернуться и пройти заново неправильные

#### 6. ✏️ **Пробелы в мантрах** (Модули 3-11) - *Особая логика*
- **Разделение на 3 временные формы**:
  - **Präsens**: 1/3 от прогресса раздела
  - **Präteritum**: 1/3 от прогресса раздела
  - **Partizip II**: 1/3 от прогресса раздела
- **Прогресс каждой формы**: (Количество правильных) / (Всего упражнений) × 100%
- **Как работает**:
  - ✅ Правильный ответ → сохраняется для конкретного упражнения в конкретном времени
  - ❌ Неправильный ответ → сохраняется как ошибка
  - **Прогресс отображается постепенно** для каждого времени
  - **Формула итогового прогресса раздела**:
    ```
    Прогресс = (Präsens % правильных × 33.3%) +
               (Präteritum % правильных × 33.3%) +
               (Partizip II % правильных × 33.3%)
    ```
  - **Пример**: Präsens 80%, Präteritum 60%, Partizip II 100% → Прогресс раздела = (80%×33.3% + 60%×33.3% + 100%×33.3%) = **80%**
- **Правило**: 
  - Неправильные ответы НЕ засчитываются в прогресс
  - Можно возвращаться и проходить только неправильные/непройденные
  - Галочка ✅ появляется только при 100% (все три формы полностью правильно)

#### 7. 📝 **Пробелы в тексте**
- **Условие завершения**: ✅ **Все пропуски заполнены ПРАВИЛЬНО**
- **Прогресс**: (Количество правильных ответов) / (Всего пропусков) × 100%
- **Правило**: 
  - Выбор из выпадающего списка
  - Неправильные ответы НЕ засчитываются
  - Можно пересдать

#### 8. ⌨️ **Заполнение слов** (Typing Trainer)
- **Условие завершения**: ✅ **Все три формы глагола введены ПРАВИЛЬНО**
- **Прогресс**: (Количество полностью правильно заполненных глаголов) / (Всего глаголов) × 100%
- **Правило**:
  - Нужно правильно ввести: Präsens, Präteritum, Partizip II
  - Глагол засчитывается только если все три формы верны
  - Можно исправлять после проверки

## 🔁 Интервальное повторение (ПОВТОР)

### Что это?
Система умного повторения карточек на основе фиксированных интервалов.

### Откуда берутся карточки?
- Автоматически добавляются после **завершения раздела "Карточки в 3-х формах"** в модулях 2-11
- Карточки добавляются только из модулей, которые вы прошли

### Формат карточек
- **Модуль 2**: `infinitiv, er/sie/es Präsens` (например, `dürfen, er/sie/es darf`)
- **Модули 3-11**: `infinitiv - Präteritum - Partizip II` (например, `fahren - fuhr - gefahren`)

### Алгоритм интервалов

**Фиксированные интервалы повторений:**
```
Правильный ответ №1: Сразу (0 дней)
Правильный ответ №2: Завтра (1 день)
Правильный ответ №3: Через 1 день
Правильный ответ №4: Через 3 дня
Правильный ответ №5: Через 7 дней
Правильный ответ №6: Через 14 дней
Правильный ответ №7: Через 30 дней
Правильный ответ №8: Через 90 дней
```

### Как работает?
1. **"Знаю"** ✅ - карточка перемещается на следующий интервал
2. **"Не знаю"** ❌ - карточка возвращается в начало (доступна сразу для повторения)

### Счётчик карточек
- **На сегодня**: Карточки, которые готовы для повторения прямо сейчас
- **Всего**: Все карточки, которые вы добавили в систему повторения

## 🧪 КВИЗ

### Как работает квиз?
- Случайные вопросы по глаголам из базы
- Можно выбрать: 5, 10 или 20 вопросов
- ~3 вопроса на каждый глагол (Präteritum, Partizip II, перевод)
- 3 варианта ответа на каждый вопрос

### Результаты
- ✅ Правильные ответы: отображается вопрос и правильный ответ
- ❌ Неправильные ответы: отображается вопрос, ваш ответ и правильный ответ
- Результаты сохраняются в Firebase

### Влияние на прогресс
- Квиз **НЕ влияет** на прогресс модулей
- Это отдельный инструмент для самопроверки

## ❓ Частые вопросы (FAQ)

### "Прогресс не сохраняется в упражнениях!"

**Ответ**: Прогресс **СОХРАНЯЕТСЯ**, но работает иначе, чем вы ожидаете:

#### Что сохраняется:
✅ **Детальный прогресс** для каждого упражнения:
- Правильный ответ → 🟢 зелёная отметка
- Неправильный ответ → 🔴 красная отметка
- Этот прогресс **виден на прогресс-баре** раздела

#### Что НЕ сохраняется как "завершение":
❌ **Раздел не считается завершенным**, пока не достигнут **100% правильных**:
- 80% правильных → прогресс-бар показывает 80%, но **нет галочки** ✅
- Модуль не получает очки за этот раздел
- Нужно исправить все ошибки и пройти заново

#### Как это выглядит:

```
Модуль 1: 75% прогресс
├─ Видео ✅ (завершено)
├─ Мантры ✅ (завершено)
├─ Пробелы в мантрах ⬜ (12/15 правильных = 80%, НЕ завершено!)
└─ Пробелы в тексте ✅ (завершено)

Прогресс модуля = (100% + 100% + 0% + 100%) / 4 = 75%
                               ↑
                        Этот раздел не завершён!
```

### "Я прошёл все упражнения, но прогресс 0%!"

**Причина**: Есть неправильные ответы.

**Решение**:
1. Зайдите в раздел "Пробелы в мантрах" снова
2. Система предложит: **"Пройти заново"** или **"Только неправильные"**
3. Выберите "Только неправильные"
4. Пройдите только ошибочные упражнения правильно
5. Когда все станут зелёными → раздел завершится (100%)

### "Прогресс-бар показывает 90%, но раздел не завершён!"

**Это нормально!** Раздел завершается только при **100%** правильных.

**Пример**:
- 9 из 10 упражнений правильно = 90% на прогресс-баре
- Но раздел **НЕ завершён** → нет галочки ✅
- Модуль **НЕ получает** очки за этот раздел
- Нужно пройти 10-е упражнение правильно → раздел завершится

## 🔧 Техническая информация (для администраторов)

### Как отслеживается завершение разделов?

#### Видео (Ассоциации, История банды)
```typescript
// Прогресс обновляется СРАЗУ при клике на карточку раздела
onClick={() => {
  setActiveModule('video');
  updateProgress(topicId, 'video'); // ← Отметка о завершении
}}
```
- ⚠️ **Нет проверки** факта просмотра до конца
- Iframe от Boomstream не отправляет события о завершении просмотра
- Решение: простота UX важнее точности отслеживания

#### Карточки
```typescript
<Flashcards 
  topic={selectedTopic} 
  onComplete={() => updateProgress(topicId, 'flashcards')} 
/>
```
- ✅ Колбэк `onComplete` вызывается **после просмотра всех карточек**
- Пользователь должен дойти до конца колоды

#### Упражнения (Пробелы в мантрах)
```typescript
<MantraGapExercises 
  topic={selectedTopic} 
  onComplete={() => updateProgress(topicId, 'exercises')}
  onItemProgressUpdate={updateItemProgress}
/>
```

**Модули 1-2:**
- ✅ Колбэк `onComplete` вызывается **только когда ВСЕ упражнения правильные**
  ```typescript
  const allExercisesComplete = allExercises.every((_, idx) => 
    itemProgress[idx] === true
  );
  if (allExercisesComplete && onComplete) {
    onComplete(); // ← Раздел завершён!
  }
  ```
- Расчёт прогресса:
  ```typescript
  const correctCount = Object.values(itemProgress).filter(Boolean).length;
  const percentage = (correctCount / totalItems) * 100; // 30%, 50%, 80% и т.д.
  ```
- **Важно**: Прогресс **отображается** постепенно, но раздел **не завершается** пока не 100%

**Модули 3-11:**
- Прогресс для каждого времени рассчитывается отдельно:
  ```typescript
  const praesensCorrect = itemProgress.filter(id => 
    id.includes('praesens') && value === true
  ).length;
  const praesensPercentage = (praesensCorrect / totalItems) * 100;
  
  // Аналогично для präteritum и partizip2
  
  const totalProgress = 
    (praesensPercentage × 33.3%) +
    (praeteritumPercentage × 33.3%) +
    (partizip2Percentage × 33.3%);
  ```
- Галочка ✅ появляется только при 100% (все три формы полностью правильно)

### Где хранится прогресс?
1. **Firebase Firestore** (основное хранилище):
   ```
   users/{telegramId}/progress/{topicId} → string[]
   users/{telegramId}/itemProgress/{topicId}/{moduleId}/{itemIndex} → boolean
   ```
2. **localStorage** (резервное хранилище):
   - Используется при отсутствии подключения к Firebase
   - Ключи: `progress_${telegramId}`, `itemProgress_${telegramId}`

### Формула расчета прогресса модуля
```typescript
const completedSections = sections.filter(s => s.isCompleted).length;
const moduleProgress = (completedSections / totalSections) * 100;
```

Для "Пробелы в мантрах" (модули 3-11):
```typescript
const praesensProgress = allPraesensCorrect ? 33.3 : 0;
const praeteritumProgress = allPraeteritumCorrect ? 33.3 : 0;
const partizipProgress = allPartizipCorrect ? 33.3 : 0;
const sectionProgress = praesensProgress + praeteritumProgress + partizipProgress;
```

#### Мантры
```typescript
<Mantras 
  topic={selectedTopic} 
  onComplete={() => updateProgress(topicId, 'mantras')}
/>
```
- ✅ Колбэк `onComplete` вызывается **только когда ВСЕ мантры отмечены как "Знаю"**
- Проверка:
  ```typescript
  const allCorrect = activeIndices.every(index => results[index] === true);
  if (onComplete && allCorrect) {
    onComplete();
  }
  ```
- Кнопки:
  - **"Трудно"** → `markResult(false)` (не засчитывается)
  - **"Знаю"** → `markResult(true)` (засчитывается)
- Аудио 🔊:
  - Использует OpenAI TTS с кэшированием
  - Воспроизведение **не обязательно** для завершения
  - Просто удобная функция для пользователя
