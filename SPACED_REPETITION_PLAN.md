## 🎯 Алгоритм работы:

1. **Инициализация:**
   - Загружаем все карточки глаголов из всех модулей
   - Загружаем данные интервального повторения пользователя из Firebase
   - Определяем карточки, готовые к повторению (nextReview <= now)

2. **Показ карточек:**
   - Сначала показываем карточки, готовые к повторению (dueToday)
   - Затем добавляем новые карточки (максимум 10 новых за сессию)
   - Перемешиваем порядок для разнообразия

3. **Обработка ответа:**
   - Пользователь нажимает "Знаю" или "Не знаю"
   - Обновляем данные карточки через `SpacedRepetitionSM2.update()`
   - Сохраняем обновленные данные в Firebase

4. **Следующее повторение:**
   - При правильном ответе: увеличиваем интервал (1 день → 6 дней → и т.д.)
   - При неправильном ответе: сбрасываем интервал (повтор через 10 минут)
   - Увеличиваем/уменьшаем Ease Factor

## 🔄 Интервалы повторения (SM-2):

- **1-е правильное повторение**: 1 день
- **2-е правильное повторение**: 6 дней
- **3-е и далее**: предыдущий интервал × easeFactor (обычно ~15, ~30, ~60 дней...)

## 📈 Размер данных в Firebase:

- Для 200 карточек: ~200 объектов × ~200 байт = **~40 KB на пользователя**
- Для 500 пользователей: 500 × 40 KB = **~20 MB** (очень мало!)

## ✅ Преимущества:

1. **Эффективное обучение**: Повторение только когда нужно (когда забывается)
2. **Персонализация**: Каждый пользователь имеет свой график повторений
3. **Автоматизация**: Система сама определяет, когда показывать карточку
4. **Масштабируемость**: Легко добавить больше карточек без изменения алгоритма
