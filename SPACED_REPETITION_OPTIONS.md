# Варианты организации интервального повторения для карточек глаголов

## 📊 Текущая ситуация:

- **11 модулей** с карточками глаголов
- **~20 карточек** в каждом модуле
- **~200 карточек** всего
- **Карточки добавляются постепенно** (1 модуль в день = 11 дней)

## 🔄 Вариант 1: Интервальное повторение прямо в модуле

### Как работает:
- Пользователь проходит модуль → карточки глаголов добавляются в его личную коллекцию повторений
- В модуле есть раздел "Интервальное повторение" с карточками только этого модуля
- Когда наступает время повторения, карточки показываются в модуле

### ✅ Преимущества:
1. **Контекст модуля**: Карточки остаются привязаны к модулю (логическая связь)
2. **Простота навигации**: Пользователь видит карточки там, где их изучил
3. **Меньше изменений в коде**: Не нужно создавать отдельное меню

### ❌ Недостатки:
1. **Разбросанность**: Карточки разбросаны по 11 модулям - нужно заходить в каждый модуль
2. **Сложность отслеживания**: Не видно общую статистику по всем карточкам
3. **Низкая эффективность**: Если нужно повторить карточки из разных модулей, придется переключаться между модулями
4. **Проблема с интервалами**: Карточка из модуля 1 может требовать повторения, когда пользователь изучает модуль 5 - карточка "спрятана" в другом модуле

---

## 🔄 Вариант 2: Отдельное меню "Интервальное повторение"

### Как работает:
- Пользователь проходит модуль → карточки глаголов **автоматически добавляются** в общую коллекцию повторений
- На главном экране есть отдельная кнопка/меню **"Интервальное повторение"**
- Все карточки из всех модулей собраны в одном месте
- Система показывает только карточки, готовые к повторению (с учетом интервалов)

### ✅ Преимущества:
1. **Единое место**: Все карточки в одном месте - удобно для ежедневных повторений
2. **Умный алгоритм**: Система показывает только те карточки, которые нужно повторить **прямо сейчас**
3. **Эффективность**: За одну сессию можно повторить карточки из разных модулей
4. **Статистика**: Видна общая статистика по всем карточкам:
   - Сколько карточек готово к повторению
   - Сколько новых карточек (из новых модулей)
   - Прогресс освоения (интервал > 30 дней = освоено)
5. **Гибкость**: Можно повторить карточки из модулей, которые уже давно пройдены
6. **Мотивация**: Видно общий прогресс и количество освоенных карточек

### ❌ Недостатки:
1. **Нужно создать новое меню**: Дополнительная работа по интеграции
2. **Контекст модуля теряется**: Карточки отделены от модуля, где они изучались

---

## 🎯 **РЕКОМЕНДАЦИЯ: Вариант 2 (Отдельное меню)**

### Почему Вариант 2 лучше:

1. **Интервальное повторение требует централизованного подхода**:
   - Карточки из модуля 1 могут требовать повторения, когда пользователь на модуле 7
   - С отдельным меню пользователь видит ВСЕ карточки, готовые к повторению, в одном месте

2. **Эффективность обучения**:
   - Ежедневная рутина: открыл "Интервальное повторение" → повторил все готовые карточки
   - Не нужно искать карточки по модулям

3. **Гибкость добавления карточек**:
   - Карточки автоматически добавляются из модуля в коллекцию повторений
   - Модуль можно пройти один раз, но повторения продолжаются в отдельном меню

4. **Статистика и мотивация**:
   - "Сегодня к повторению: 12 карточек"
   - "Освоено: 45 из 200 карточек"
   - "Новых карточек: 3" (из нового модуля)

## 📋 Реализация Варианта 2:

### 1. Структура данных в Firebase:

```typescript
spacedRepetition: {
  "fahren-card-forms": {
    cardId: "fahren-card-forms",
    moduleId: "module-3", // Из какого модуля карточка
    addedAt: 1704038400000, // Когда добавлена (при прохождении модуля)
    nextReview: 1704124800000, // Когда повторить
    easeFactor: 2.5,
    interval: 6,
    repetitions: 2,
    lastReview: 1704038400000,
    successCount: 3,
    failCount: 0
  },
  // ... другие карточки
}
```

### 2. Логика добавления карточек:

**Автоматическое добавление при прохождении модуля:**
- Когда пользователь проходит модуль (например, завершает `verb-form-flashcards`)
- Все `verbFormCards` из этого модуля **автоматически добавляются** в `spacedRepetition`
- Карточки получают начальные значения (готовы к повторению через 1 день)

**Пример кода:**
```typescript
// После завершения verb-form-flashcards в модуле
const moduleCards = topic.verbFormCards || [];
for (const card of moduleCards) {
  // Инициализируем карточку для интервального повторения
  if (!spacedRepetition[card.id]) {
    spacedRepetition[card.id] = SpacedRepetitionSM2.initialize(card.id);
    spacedRepetition[card.id].moduleId = topic.id; // Сохраняем модуль
    spacedRepetition[card.id].addedAt = Date.now();
  }
}
await ProgressService.saveSpacedRepetition(telegramId, name, spacedRepetition);
```

### 3. Меню на главном экране:

**Добавить кнопку "Интервальное повторение":**
- Рядом с модулями (например, вверху экрана)
- Или как отдельный модуль в сетке модулей
- Показывать бейдж с количеством карточек, готовых к повторению: "12 🔔"

### 4. Компонент `SpacedRepetitionFlashcards`:

**Что показывает:**
- Только карточки, готовые к повторению (`nextReview <= now`)
- Сначала карточки, готовые к повторению
- Затем новые карточки (из новых модулей) - максимум 10 новых за сессию
- Статистику: "К повторению: 12 | Новых: 3 | Всего освоено: 45/200"

**Как работает:**
1. Загружает все карточки из всех модулей (из `TOPICS`)
2. Загружает `spacedRepetition` из Firebase
3. Определяет карточки, готовые к повторению (алгоритм SM-2)
4. Показывает их по одной с возможностью ответить "Знаю" / "Не знаю"
5. Обновляет данные карточки и сохраняет в Firebase

## 🔄 Гибридный вариант (опционально):

**Можно объединить оба подхода:**
- Отдельное меню для ежедневных повторений (Вариант 2)
- В модуле показывать статус карточек этого модуля (для контекста)
- Например: "В этом модуле: 20 карточек | К повторению: 3"

## 📊 Размер данных в Firebase (Вариант 2):

Для 1 пользователя с 200 карточками:
- 200 карточек × ~200 байт = **~40 KB**
- Для 500 пользователей: 500 × 40 KB = **~20 MB** ✅

## ✅ Вывод:

**Вариант 2 (Отдельное меню) - лучший выбор**, потому что:
1. ✅ Соответствует принципам интервального повторения (централизованный доступ)
2. ✅ Эффективнее для ежедневных повторений
3. ✅ Лучшая статистика и мотивация
4. ✅ Гибкость (карточки доступны даже после прохождения модуля)
5. ✅ Масштабируемость (легко добавить больше карточек)

**Карточки автоматически добавляются из модуля при его прохождении**, но повторения происходят в отдельном централизованном меню.
